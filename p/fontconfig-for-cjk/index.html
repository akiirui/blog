CJK 用户们的 Fontconfig

让字体显示更合你心意。

Mar 26, 2021

起因

由于汉字在东亚地区的广泛使用，以及各个地区的文化发展，同一个汉字可能在不同的地区有着不同字形。

在没有配置 Fontconfig 的情况下，可能会出现简体中文的汉字错误的显示为了日语汉字字形的问题。

这个问题在 Windows 下近乎无解。但是在 Linux 下，我们可以自由的配置 Fontconfig 在系统层面几乎完美的解决这个问题。

需求

我个人的需求是，拉丁字符使用特定的字体。而 CJK 字符则按照语言选择所需的字形。

于是要优先使用拉丁字体。如果某个字符不在其中，则使用后备的 CJK 字体。

字体

在这里，我使用了以下这些字体来进行配置。当然，你也可以选择你喜欢的字体。

CharsetSerifSansMono
LatinPT SerifRobotoCascadia Code
CJKNoto Serif CJKNoto Sans CJKNoto Sans Mono CJK

还有 Noto Color Emoji 作为 Emoji 字体。

配置

由于完整配置太长，为了文章的简洁，我只简单介绍每一个文件是如何配置的，以便于各位使用时按需自定义。

完整配置储存在 Gist > Customize Fontconfig 。

拉丁字符 52-latin.conf

<alias binding="strong">
  <family>serif</family>
  <prefer>
    <family>PT Serif</family>
  </prefer>
</alias>

在拉丁字符部分，使用 alias & prefer 使所需的字体成为第一个匹配项。

binding="strong" 是为了使这条规则不会被顺序更靠后的配置文件给覆盖。

family 的值则按照需要更改为 sans-serif monospace system-ui 。

CJK 字符 53-cjk.conf

<match target="pattern">
  <test name="family">
    <string>serif</string>
  </test>
  <edit name="family" mode="append" binding="strong">
    <string>Noto Serif CJK SC</string>
  </edit>
</match>

在 CJK 字符部分，使用 match & test 匹配字体家族，然后用 edit mode="append" 使所需的字体成为紧随拉丁字体的后备字体。

这一条规则是当没有提供 lang 属性的时候的默认值，按需更改字形变体。

<!-- "lang=zh-hant" or "lang=zh-hant-*" -->
<match target="pattern">
  <test name="lang" compare="contains">
    <string>zh-hant</string>
  </test>
  <test name="family">
    <string>serif</string>
  </test>
  <edit name="family" mode="append" binding="strong">
    <string>Noto Serif CJK TC</string>
  </edit>
</match>

此规则是当提供了 lang 属性时选择的字形变体，匹配 zh-hant 或 zh-hant-* （忽略大小写）。

放置在后面则可以覆盖上面所配置的 CJK 字符默认值。

常见的 lang 属性

ChineseJapaneseKorean
zhjako
zh-CNja-jpko-kr
zh-TW
zh-HK
zh-Hans
zh-Hant
zh-Hant-HK

Emoji 😆 54-emoji.conf

<match target="pattern">
  <test name="family">
    <string>serif</string>
  </test>
  <edit name="family" mode="append_last" binding="strong">
    <string>Noto Color Emoji</string>
  </edit>
</match>

这条规则是将 Emoji 字体放置于后备字体列表的末尾。当然，它依然优先于其他没有在配置文件中列出的字体。

最后

虽然配置 Fontconfig 可以几乎完美的达成需求，但是在某些不遵循 Fontconfig 的应用程序中就无可奈何了。

Chromium 系

Chromium 只在 UI 字体上遵循 Fontconfig，并且 CJK 的后备字体是硬编码的。你只有使用 Advanced Font Settings 来设置每种语言标签的字体。

还好 Linux 下大多数应用都遵循了 Fontconfig 。立即转投 Firefox 吧！
